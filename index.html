The Problem

A Shopify clothing brand averaging 80+ orders/day was losing roughly 65% of customers at checkout. Their only recovery tool was a single email sent 24 hours later — by which time most customers had moved on. The founder was manually following up on high-value carts via WhatsApp, which was unsustainable and slow.

The Solution

Built a fully automated, AI-powered cart recovery system that triggers within 15 minutes of abandonment. It sends a personalised WhatsApp message using the customer's name and exact abandoned products, follows up twice more at strategic intervals, and handles customer objections conversationally — no human needed at any step.

Workflow Breakdown

Trigger
Shopify webhook fires on cart abandonment event
→
Validation
Checks: phone number present, cart value > ₹500, not already messaged
→
AI Logic
GPT-4 generates personalised message with product names & urgency tone
→
Action
WhatsApp message sent via Meta Cloud API with direct checkout link
→
Fallback
If WhatsApp fails → email sent automatically via SMTP
→
Logging
All events stored in Google Sheets dashboard for monitoring

Reliability & Error Handling

If WhatsApp API is down, system automatically falls back to email — zero missed follow-ups
If phone number is missing or invalid, cart is flagged and skipped without crashing the system
Automatic retry logic: if delivery fails, it retries 3 times before switching to fallback
Every event logged with timestamp, status, and error code for full visibility and debugging

The Problem

A B2B services company was receiving 30–50 leads per day through their website. Every lead had to be manually read, categorised, entered into the CRM, and followed up on — a process that took 5+ hours daily and was riddled with errors and delays. Hot leads were going cold while the team was buried in admin work.

The Solution

Built an end-to-end automated lead pipeline. Every incoming enquiry is processed by AI which scores lead quality, extracts key details, auto-populates the CRM, and triggers the appropriate follow-up — all within 60 seconds of form submission. The sales team only sees qualified, pre-scored leads with a full summary ready.

Workflow Breakdown

Trigger
Form submission webhook fires on lead capture
→
Validation
Checks for required fields, filters spam, deduplicates existing contacts
→
AI Scoring
GPT-4 reads the enquiry and assigns Hot / Warm / Cold with reasoning
→
Action
CRM auto-updated + Slack alert sent to sales team with full AI summary
→
Fallback
If CRM API fails → lead stored in Google Sheets and team alerted via email
→
Logging
Full audit trail: lead score, processing time, actions taken, errors if any

Reliability & Error Handling

If CRM API is unavailable, leads are safely stored in Google Sheets — nothing is ever lost
Duplicate detection prevents the same lead being entered twice across channels
If AI scoring fails, lead is flagged for manual review instead of being silently skipped
Daily summary report sent to manager with total leads, scores breakdown, and any errors

🛡 Why fallback systems matter

External APIs — WhatsApp, Shopify, CRMs — go down. Without a fallback, your system silently fails and you lose data. Every system I build has a backup path so nothing is ever dropped, even when third-party services have outages.

📋 Why logging matters

You can't fix what you can't see. Every automation I build writes a log of what it did, when, and whether it succeeded. This means bugs are caught fast, performance is measurable, and you always have proof of every action taken.

✅ Why validation matters

Bad data in = broken system out. Before any action runs, I validate the incoming data — checking for missing fields, invalid formats, and edge cases. This prevents crashes, errors, and incorrect actions that would embarrass your business.
